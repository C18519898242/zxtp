# AI Context 资产负债摘要

## 背景

AI Context 的“资产与负债”只有占位说明。资产负债表已写入 `financial_balance_sheets`，但其原始字段此前未映射为可读指标。

## 验证过的字段

对 `600011` 的 `zcfzb` 缓存进行跨报告期验证后，以下字段在 2004 年以来的 93 个可用报告期满足 `资产总计 = 负债合计 + 所有者权益合计`：

| 展示项目 | 原始字段 |
| --- | --- |
| 资产总计 | `T039` |
| 负债合计 | `T062` |
| 所有者权益合计 | `T071` |

2001–2003 年的 8 个历史报告期不满足该恒等式；AI Context 只展示最近三个完整年度和最新报告期，不使用这些历史期。

## 设计

在 `render_financial_analysis()` 中新增资产负债摘要渲染：

1. 复用 `financial_context_periods()` 从关键指标选出的报告期与现有列标题；
2. 从 `financial_balance_sheets` 查询同一股票、同一报告期的 `T039`、`T062`、`T071`；
3. 输出“资产总计”“负债合计”“所有者权益合计”，金额由元转换为亿元并保留两位小数；
4. 输出“资产负债率（%）”，按 `T062 / T039 × 100` 计算并保留两位小数；
5. 不存在的值显示 `—`；资产总计为零或缺失时，资产负债率显示 `—`；
6. 不展示货币资金或其他未验证原始字段。

“资产与负债”表与“经营成果与关键指标”使用相同的报告期列，从而便于横向阅读。

## 测试

为 AI Context 合成一组包含关键指标和三项资产负债字段的数据，断言：

- 表格使用既有的三年年报加最新报告期；
- 三项总额按亿元格式化；
- 资产负债率计算正确；
- 缺失值与零资产分母显示 `—`。

## 非目标

- 不修改 DuckDB schema 或 raw 解析器；
- 不解释或展示未验证的 `zcfzb` 字段；
- 不从早于 2004 年的异常报告期推导指标；
- 不改变利润表和现金流量表的展示范围。
