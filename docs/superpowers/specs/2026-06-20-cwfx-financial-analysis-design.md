# 财务分析（cwfx）结构化数据与 AI Context 设计

**日期：** 2026-06-20
**范围：** 第一阶段仅覆盖核心财报三表与关键财务指标；默认呈现最近三个完整年度和最新报告期。

## 目标

将已缓存的 `cwfx` 原始 JSON 转换为可查询、可追溯的 DuckDB 数据，并在单股票 `full_context.md` 的“3. 财务分析”章节输出适合基本面分析的事实摘要。

本阶段提供可比的经营成果、资产负债、现金流和关键效率/盈利指标；不生成买卖建议，也不把未确认口径的原始字段解释为财务结论。

## 数据范围

解析器读取下列已由 `fetch-cwfx` 缓存的模块：

| 模块 | 数据 | 输出表 |
| --- | --- | --- |
| `lyb` | 利润表 | `financial_income_statements` |
| `zcfzb` | 资产负债表 | `financial_balance_sheets` |
| `xjllb` | 现金流量表 | `financial_cash_flows` |
| `zyzb` | 主要财务指标 | `financial_key_metrics` |

`cwbg` 及其余 `cwfx` 模块继续作为 raw evidence 保留，不纳入第一阶段结构化解析。后续模块扩展不得改变上述表的既有列语义。

## 结构化设计

新增 `parse_financial_analysis(stock_code, data_root)`，从 raw cache 中读取上述模块，创建或更新 `data_root/warehouse/financial.duckdb`。

每张表都至少保存：

- `stock_code`
- `report_date`
- `report_year`
- `report_type`（年报/中报/一季报/三季报等原始标识的规范化值）
- `statement_scope`（合并、母公司或 raw 未知）
- `source_module`
- `raw_field_name`（适用于按指标存储的字段）

三张报表保存项目级事实：标准项目名、金额（元）和可用的同比/期初比较字段。关键指标表保存标准指标名、数值、单位以及原始字段名。标准字段只在已确认其来源口径时写入；不能确认的字段不猜测映射。

写入为幂等替换：相同股票、报告期、报表范围和标准项目/指标再次解析时，由最新 raw snapshot 覆盖旧记录，避免重复行。空的、缺失的或无法读取的 raw 文件不阻止其余模块写入。

## CLI 集成与故障处理

`fetch-cwfx` 在所有下载任务结束后调用解析器，输出 DuckDB 文件位置，方式与现有公司概况和研报评级的结构化步骤一致。

当个别子模块下载失败时，保留当前抓取命令的失败重试与日志行为；解析器只处理本地实际存在、格式可读的快照。JSON 缺少结果集、字段或行时创建所需表但不写入虚假数据。

## AI Context 设计

`generate_full_context()` 读取 `financial.duckdb`，以固定的第 3 章小节输出：

1. **报告期说明**：说明选取最近三个完整年度及最新报告期，并标注报告期、报表范围和数据来源。
2. **经营成果**：营收、营业利润（若可用）、归母净利润（若可用）、毛利率、净利率、ROE 等经确认指标的跨期表格。
3. **资产与负债**：总资产、总负债、归母权益、资产负债率及流动性指标（仅在对应字段已确认时）。
4. **现金流与效率**：经营/投资/筹资活动现金流净额、自由现金流或周转类指标（仅在语义和计算口径已明确时）。

金额以亿元显示，原始数据仍以元保存在 DuckDB。每个小节只渲染已确认、至少存在一个值的项目；`NULL` 显示为 `—`。不从累计季报擅自推导单季度值，也不在未明确计算规则时派生自由现金流。

如果 DuckDB 不存在、不可读或该股票没有结构化记录，第 3 章保留稳定的降级说明，并列出可追溯的 raw entry/module。其他 AI Context 章节必须继续生成。

## 测试与验收

测试使用合成 raw JSON，覆盖：

- 四个模块分别创建正确表和规范字段；
- 数值转换、报告期/报表类型识别与 raw 字段追溯；
- 重复解析对相同业务键的替换；
- raw 缺失、空结果集、局部模块损坏时的可恢复行为；
- `fetch-cwfx` 完成后调用结构化解析；
- AI Context 只保留三年年报和最新报告期，金额显示、缺失值和降级提示正确；
- Context 不将未确认口径呈现为标准财务结论。

## 非目标

- 不在本阶段结构化全部 `cwfx` 子模块；
- 不抓取或解析财报 PDF；
- 不提供自动投资评级、预测或买卖建议；
- 不变更 raw cache 的目录组织和保存策略。
