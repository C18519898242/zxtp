# 盈利预测年度指标设计

## 目标

将已确认的 `ybpj/ylyctj` 盈利预测字段汇总为一张按年度阅读的 DuckDB 指标表，并在 AI Context 展示历史三年与预测三年的对照表。

## 数据模型

保留现有五张 `earnings_forecast_*` raw 表不变，新增派生表 `earnings_forecast_yearly_metrics`。一行表示一只股票、一个财年的一组已确认指标：

| 字段 | 含义 | 单位 |
| --- | --- | --- |
| `stock_code` | 六位股票代码 | - |
| `fiscal_year` | 财年 | 年份 |
| `period_type` | `actual` 或 `forecast` | - |
| `earnings_per_share` | 每股收益 | 元 |
| `book_value_per_share` | 每股净资产 | 元 |
| `return_on_equity_pct` | 净资产收益率 | % |
| `net_profit_parent_wan` | 归属母公司净利润 | 万元 |
| `net_profit_growth_pct` | 归母净利润同比增长率 | % |
| `operating_revenue_wan` | 营业收入 | 万元 |
| `operating_profit_wan` | 营业利润 | 万元 |

主键为 `stock_code + fiscal_year`。表与 raw 表使用同一份 `ylyctj` 来源元数据；重新下载该股票时在同一事务内整体替换其年度指标记录。

## 字段映射

历史实际数据来自结果集 3：

| 指标 | TQLEX 字段 |
| --- | --- |
| 财年 | `T002` |
| 每股收益 | `T055` |
| 每股净资产 | `T059` |
| 净资产收益率 | `T064` |
| 归属母公司净利润 | `T018` |
| 归母净利润增长率 | `T118` |
| 营业收入 | `T003` |
| 营业利润 | `T012` |

预测数据来自结果集 1 和结果集 2。结果集 1 的 `nyear` 是预测起始年份；结果集 2 的三列依次对应起始年份、起始年份加一、起始年份加二：

| 指标 | 预测第 1 / 2 / 3 年 TQLEX 字段 |
| --- | --- |
| 每股收益 | `T036` / `T037` / `T038` |
| 每股净资产 | `T027` / `T028` / `T029` |
| 净资产收益率 | `T024` / `T025` / `T026` |
| 归属母公司净利润 | `T033` / `T034` / `T035` |
| 营业收入 | `T021` / `T022` / `T023` |
| 营业利润 | `T030` / `T031` / `T032` |

预测归母净利润增长率由相邻年度的 `net_profit_parent_wan` 计算：`(当年 / 上年 - 1) * 100`。首个预测年度以最近一个实际年度为上年；若上年净利润缺失或为零，则该值为 `NULL`。

## AI Context

“盈利预测统计”章节改为显示：预测起始年度、来源日期、机构覆盖数量，以及一个年度列、指标行的 Markdown 表格。金额列从万元换算为亿元并显示两位小数；每股和百分比指标显示三位或两位小数。表格明确标记“实际”与“预测”。

市盈率不进入派生表或 AI Context。当前 `ylyctj` raw 中没有可可靠复现截图市盈率的直接字段；用当前价格除以预测每股收益得到的结果与来源页面不一致，因此不猜测口径。

## 容错与测试

- `ylyctj` 缺失时，不改动既有年度指标；空结果集只清空该股票的年度指标并保留其他 ybpj 模块。
- 缺少字段、无法解析为数值、或预测起始年份无效时，对应指标保存为 `NULL`，不让解析失败。
- 测试覆盖历史与预测字段映射、预测增长率计算、同股票重新解析替换、AI Markdown 六年表格、以及旧数据库缺少派生表时的可读降级。
