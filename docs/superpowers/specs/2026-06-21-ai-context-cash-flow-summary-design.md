# AI Context 现金流摘要

## 背景

AI Context 的“现金流与效率”只有占位说明。现金流量表已写入 `financial_cash_flows`，但其原始字段尚未映射为可读的现金流摘要。

## 验证过的字段

对 `600011` 的 `xjllb` 缓存进行跨报告期验证后，以下字段可用于现金流摘要：

| 展示项目 | 原始字段 |
| --- | --- |
| 经营活动产生的现金流量净额 | `T017` |
| 投资活动产生的现金流量净额 | `T029` |
| 筹资活动产生的现金流量净额 | `T038` |
| 汇率变动对现金及现金等价物的影响 | `T039` |
| 现金及现金等价物净增加额 | `T041` |
| 期初现金及现金等价物余额 | `T042` |
| 期末现金及现金等价物余额 | `T043` |

在 99 个可用报告期中，`T041 = T017 + T029 + T038 + T039` 全部成立。`T043 = T042 + T041` 在 99 个可用报告期中有 98 个成立；AI Context 只展示最近三个完整年度和最新报告期。

## 设计

在 `render_financial_analysis()` 中新增现金流摘要渲染：

1. 复用关键指标选出的报告期与列标题；
2. 从 `financial_cash_flows` 查询以上七个字段；
3. 按表格顺序输出七项，金额由元转换为亿元并保留两位小数；
4. 任何缺失值显示 `—`；
5. 不推导自由现金流、经营现金净额/净利润比或周转率；
6. 保留现有“每股经营现金流（元）”于“经营成果与关键指标”表。

## 测试

为 AI Context 合成现金流原始数据，并断言：

- 七项现金流与现金余额按既有四个报告期渲染；
- 金额按亿元格式化，负值保留负号；
- 缺失原始值显示 `—`；
- 旧的占位说明不再出现。

## 非目标

- 不修改 DuckDB schema 或现金流 raw 解析器；
- 不解释未验证的 `xjllb` 原始字段；
- 不派生自由现金流、现金转换率或周转率；
- 不更改资产负债摘要或经营成果表。
