# 财务关键指标报告期解析修复

## 背景

`cwfx/zyzb` 的真实 TQLEX 响应将报告期放在 `T002` 字段。当前解析器仅从 `rq` 读取报告期，导致每一行都被跳过，`financial_key_metrics` 虽被建表但没有数据。AI Context 只读取该表，因此财务章节显示“暂无结构化财务分析”。

## 目标

确保 `zyzb` 原始响应中的 `T002` 报告期能够写入 `financial_key_metrics`，并保持对已有 `rq` 格式的兼容。

## 设计

在 `replace_financial_key_metrics()` 中，按以下顺序解析报告期：

1. 读取 `rq`；
2. 若为空，读取 `T002`；
3. 两者均为空时跳过该行，维持当前无效行处理行为。

不修改数据库结构、其他财务表的解析规则或 AI Context 的查询逻辑。写入后的 `report_date`、`report_year` 和 `report_type` 继续复用已有标准化逻辑。

## 测试

增加回归测试，使用真实 `zyzb` 的 `T002` 列名和至少一个已映射指标，验证：

- `financial_key_metrics` 产生对应记录；
- 报告期和报告类型正确；
- 既有 `rq` 格式测试继续通过。

## 非目标

- 不为其他财务模块引入通用字段别名框架；
- 不从利润表、资产负债表或现金流量表回填关键指标；
- 不修改 AI Context 的降级展示策略。
